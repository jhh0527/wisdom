# 유튜브 대본 작성·변경 가이드 (재사용)

이 문서는 **한 번 적용한 대본 변경 규칙**을 정리해 두고, **다른 주제 스크립트**를 만들 때도 같은 기준으로 쓰기 위한 메모입니다. AI에게 이 파일을 함께 주거나, 아래 **프롬프트 블록**만 복사해 사용하면 됩니다.

> **핵심 한 줄.** 단순 위키 스타일이 아니라 **넷플릭스 역사 다큐**처럼, **인간 심리와 시대 분위기**를 살리고 **장면 전환이 있는 서사형 문체**로 쓴다.

---

## 0. 톤 원칙 — 넷플릭스 역사 다큐 스타일

모든 스크립트는 아래 원칙을 **1순위**로 따른다. 이후 1·2절(아나운서체·자막 규칙)은 이 원칙을 **표현 형식**으로 다듬는 단계다.

### 0.1 무엇을 하지 않는가 (위키 회피)

* 사건을 **연·월·통계**만으로 나열하지 않는다.
* “~로 알려져 있다”, “~라고 한다” 식의 **건조한 보고체**로만 끝내지 않는다.
* 인물·집단을 **명사 카드**처럼만 소개하지 않는다 (직책·국적·생몰만 던지기 금지).
* 출처·각주를 **본문 흐름을 끊으며** 매 문장 뒤에 붙이지 않는다.

### 0.2 무엇을 하는가 (다큐 서사)

1. **장면(Scene)으로 시작한다.** 한 인물의 한순간, 한 도시의 한 풍경, 한 편지의 한 문장 등 **구체적 이미지**로 문을 연다.
2. **시대 분위기**를 한두 문장으로 깐다. 냄새·소음·불안·기대 같은 **감각·정서어**를 곁들인다.
3. **인물의 동기·두려움·욕망**을 추정 가능한 범위에서 짚는다. 단정이 어렵다면 “~했을 것이다”, “~로 보인다”로 **신중한 서술**을 사용한다.
4. **갈등→선택→결과** 순으로 흐름을 만든다. 단순 사실 나열이 아니라 **이야기의 인과**를 보여 준다.
5. **장면 전환**을 명시한다. “시간이 흐른다”, “무대는 런던으로 옮겨 간다”, “같은 시각, 파리에서는…” 같은 **시간·공간 큐**를 자주 쓴다.
6. **내레이터의 시선**을 살짝 드러낸다. 청자에게 묻거나(“이 선택을 어떻게 보아야 할까요”), 의미를 짚어 준다(“이 한 통의 편지가, 이후 100년을 바꾸게 됩니다”).
7. **반복 모티프**를 둔다. 한 가문이라면 ‘다섯 형제’, ‘편지’, ‘국경’ 같은 **상징어**를 회차 내내 다시 호출한다.

### 0.3 문장·어휘 가이드

* **현재형 사용**을 두려워하지 않는다. “1812년, 한 노인이 다섯 아들을 부른다.” 처럼 **눈앞의 장면화**.
* **감각어·정서어**를 의식적으로 섞는다. (예: 차가운, 비좁은, 흔들리는, 들끓는, 숨죽인, 어두운, 분주한)
* 짧은 문장과 긴 문장을 **교대**한다. 결정적 순간 직전엔 **짧게 끊어** 긴장감을 만든다.
* **숫자·통계**는 그 자체로 두지 말고, **체감 비유**나 **결과의 의미**와 함께 쓴다.
* **고유명사**는 첫 등장 때 **한 줄 정체성**을 곁들인다. (예: “프랑크푸르트 유대인 거주구, 좁고 어두운 골목길의 이름은 유덴가세였다.”)
* **음모론·과장**은 단정하지 않고, “이런 이야기가 자랐다 → 사실은 이렇다” 식의 **이중 시점**으로 처리한다.

### 0.4 장면 전환 표현 모음 (그대로 써도 좋음)

* “무대는 ___으로 옮겨 갑니다.”
* “시간이 ___년쯤 흐릅니다.”
* “같은 시각, ___에서는 다른 일이 벌어지고 있었습니다.”
* “여기서 한 사람을 더 만나야 합니다.”
* “장면을 잠시 ___로 돌려 보겠습니다.”
* “이야기는 다시 처음의 골목길로 돌아옵니다.”

### 0.5 회차 구조 템플릿 (권장)

1. **콜드 오픈 (0~20초)** — 한 장면·한 문장·한 이미지. 질문이나 작은 미스터리로 닫기.
2. **세계관 셋업** — 시대 분위기, 무대, 주요 인물 한두 명을 감각적으로 소개.
3. **갈등 시작** — 인물의 욕망·두려움·제약을 보여 줌.
4. **사건의 전개** — 장면 전환을 활용해 시간·공간 이동.
5. **전환점 / 클라이맥스** — 결정적 선택 또는 사건. 짧은 문장으로 박자.
6. **여진과 의미** — 결과가 다음 세대·시대에 어떻게 이어졌는지.
7. **현재로 호출** — 오늘의 우리에게 어떤 의미인지, 또는 다음 회차 미끼.

## 1. 문체 표현 규칙 (아나운서·소개 영상용)

**목표:** 0절의 서사를 **유튜브 내레이션**에 얹기 위한 표현 통일.

| 항목 | 할 일 |
| --- | --- |
| 말하기체 | `~입니다`, `~습니다`, `~하셔야 합니다` 등 방송·TTS에 맞는 종결로 통일 |
| 도입·전환 | 섹션마다 질문·안내·**장면 전환 큐**로 청자를 끌어당김 (예: “누구였을까요”, “무대는 런던으로 옮겨 갑니다”) |
| 호흡 | 긴 문장은 내레이션 단위로 끊고, 논문·에세이체는 말로 풀어 읽기 쉽게 |
| 시점 | 위키체 “~로 알려져 있다”를 줄이고, **현재형·장면형**과 혼용 (“그는 다섯 아들을 부른다.”) |
| 감정 | 인물의 **동기·심리**를 추정 표현(“~했을 것입니다”, “~처럼 느껴졌을 것입니다”)으로 살짝 얹음 |
| 편집 친화 | Markdown 굵게 등 **영상 자막에 안 쓰는 표기**는 제거하거나 본문에서 풀어 씀 |
| 출처 | 반복되는 각주·출처 나열은 줄이고, “기록에 따르면”, “아카이브에 따르면”처럼 **말로 자연스럽게**만 남김 |
| 화면 안내 | “대표 이미지 자료” 등 문서용 제목은 **“화면 자료 안내”**처럼 나레이션용으로 바꿈 |
| 내용 | **사실·구조·논지는 유지**, **톤·문장 길이·장면화만** 영상용으로 조정 (사실 변형 금지, 추정은 추정으로 표시) |

## 2. 자막용 줄바꿈 규칙 (파일 구조)

자막 편집기·SRT 입력에 맞추려면 **플레인 텍스트**에서 아래 규칙을 쓴다. (적용 파일: `로스차일드.txt`)

1. **빈 줄 한 칸** = 자막 **한 덩어리(한 큐)**의 끝. 같은 화면에 같이 뜨는 구간이다.
2. **한 덩어리 안은 최대 2줄**이 이상적이다. 위줄·아래줄이 한 큐의 1·2번째 자막 줄이 된다.
3. **섹션 제목**만 있는 줄은 한 큐로 두거나, 바로 다음 큐와 이어서 타이밍을 잡는다.
4. 한 줄이 지나치게 길면 **쉼표·절 경계**에서 끊어 두 줄로 나눈다. (모바일 가독성)
5. **장면 전환 큐**(“무대는 옮겨 갑니다…” 등)는 **앞뒤로 빈 줄을 한 칸 더** 둬 호흡을 만든다.

## 3. 다른 스크립트 생성 시 AI에게 줄 프롬프트 (복사용)

아래를 **주제만 바꿔** 복사해 사용한다. 0절(서사 원칙)을 1순위로 강조한다.

```text
역할: 넷플릭스 역사 다큐 스타일의 한국어 내레이션 작가.

톤 원칙(가장 중요):
- 단순 위키 스타일 금지. 사실 나열만 하지 말 것.
- 인간 심리(욕망·두려움·동기)와 시대 분위기(감각·정서)를 함께 그릴 것.
- 장면(Scene)으로 시작하고, 시간·공간 전환 큐를 명시할 것.
  (예: "무대는 런던으로 옮겨 갑니다", "같은 시각, 파리에서는…")
- 현재형 장면화와 과거형 설명을 자연스럽게 혼용할 것.
- 추정은 "~했을 것입니다"처럼 추정 표현으로 표시. 사실은 변형 금지.
- 통계·연도는 의미·체감 비유와 함께 쓸 것.

문체:
- 종결은 ~입니다/~습니다 위주의 아나운서 설명체.
- 짧은 문장과 긴 문장을 교대해 박자를 만들 것.
- 반복 모티프(상징어) 한두 개를 회차 전반에 다시 호출할 것.

구조 권장:
1) 콜드 오픈(한 장면) → 2) 시대 셋업 → 3) 갈등 → 4) 전개(장면 전환)
→ 5) 전환점/클라이맥스 → 6) 여진과 의미 → 7) 현재로 호출.

자막용 줄바꿈:
- 빈 줄 = 자막 한 큐, 큐당 최대 2줄.
- 장면 전환 큐는 앞뒤로 빈 줄을 한 칸 더 두어 호흡.
- 섹션 제목은 나레이션으로 읽히게 다듬을 것.

기타:
- Markdown 굵게/각주 나열은 제거하거나 말로 풀어 쓸 것.
- 출처는 "기록에 따르면", "아카이브에 따르면" 식으로 자연스럽게만.

출력: 플레인 텍스트만. 파일 맨 위에 사용 규칙(빈 줄=큐) 한 줄 안내.
```

추가로 **이 `대본작성법.md` 전체**를 컨텍스트로 함께 주면, 0~2절 규칙을 모두 따르도록 요청할 수 있다.

## 4. 최종 점검 체크리스트 (스크립트 확정 전)

**서사 (0절)**

- [ ] **첫 30초**가 “장면”으로 시작하는가 (정의·연도 나열로 시작하지 않았는가).
- [ ] 시대 **분위기·감각**을 보여 주는 문장이 섹션마다 1개 이상인가.
- [ ] 주요 인물의 **동기·두려움·욕망**이 한 번이라도 드러나는가.
- [ ] **장면 전환 큐**(“무대는 ___으로”, “같은 시각 ___에서는…”)가 충분히 있는가.
- [ ] **갈등 → 선택 → 결과**의 흐름이 살아 있는가.
- [ ] 위키식 “~로 알려져 있다” 보고체로만 채워진 단락이 없는가.
- [ ] 추정과 사실이 **표현으로 구분**되어 있는가 (사실 왜곡 없음).

**문체·자막 (1·2절)**

- [ ] TTS/아나운서체 종결이 통일됐는가 (`입니다/습니다` 혼용 난립 없음).
- [ ] 한 큐에 **3줄 이상** 붙어 있지 않은가 (가능하면 큐당 1~2줄).
- [ ] 빈 줄로 **큐 구분**이 일관되는가.
- [ ] 출처·각주가 **읽기 부담**이면 줄였는가.
- [ ] 영상에 안 나올 표기(Markdown 등)를 제거했는가.

* 자막 편집용 → **빈 줄 큐 + 큐당 최대 2줄** 구조로 `로스차일드.txt`에 반영.
