# ElevenLabs TTS 업로드 가이드 (한국어 내레이션용)

이 문서는 `로스차일드tts.txt` 같은 한국어 내레이션 원고를 **ElevenLabs**에 업로드해 음성으로 변환할 때 따르는 **파일 형식·옵션·운영 규칙**을 정리한 것이다. 새 주제 스크립트도 같은 절차로 처리한다.

> **핵심 한 줄.** 자막용 줄바꿈은 모두 풀고, **문장 부호로만 호흡**을 만든 **UTF-8 플레인 텍스트**를 올린다. 한국어는 **Multilingual v2** 또는 **Eleven v3**를 쓴다.

---

## 0. 사용 모델·언어 설정

| 항목 | 값 |
| --- | --- |
| 권장 모델 | **Eleven Multilingual v2** (안정·장문 내레이션에 강함) |
| 최신 옵션 | **Eleven v3 (alpha)** — 감정·강세 표현이 더 풍부, 오디오 태그 지원 |
| 언어 | 한국어 (Korean) — 모델이 자동 감지하지만 Studio에서는 명시 가능 |
| 출력 포맷 | MP3 44.1kHz 128kbps (기본) 또는 PCM 24kHz/44.1kHz (편집 시) |

> 유튜브 내레이션처럼 **장시간·일관된 톤**이 중요한 경우 **Multilingual v2**를 1순위로 쓰고, 한 인물의 감정 변화를 살리고 싶을 때만 v3를 시도한다.

## 1. 파일 형식 규칙 (업로드 직전 점검)

`로스차일드tts.txt`에 적용된 규칙. 새 원고도 동일하게 만든다.

1. **인코딩: UTF-8 (BOM 없음)**. PowerShell에서 만들 때 `Out-File -Encoding utf8`처럼 BOM이 붙지 않게 한다.
2. **확장자: `.txt`**. Markdown(`#`, `**`, `>`)은 모두 풀어 쓴다. 굵게·기울임은 음성에 반영되지 않는다.
3. **자막용 2줄 분할 금지.** 캡션(`.txt` 자막) 버전과 TTS 버전은 **별도 파일**로 관리한다.
   - 자막 원본: `로스차일드.txt` (큐당 1~2줄, 빈 줄=큐)
   - TTS 입력: `로스차일드tts.txt` (자연스러운 문장, 빈 줄=문단 호흡)
4. **한 문단 = 한 호흡 단위.** 문단 사이는 **빈 줄 한 칸**, 챕터 전환은 **빈 줄 두 칸** 정도가 적당.
5. **섹션 제목은 마침표로 닫는다.** 예: `다섯 아들과 유럽 네트워크.` — 그래야 TTS가 문장 끝으로 인식해 자연스러운 휴지를 둔다.
6. **이상한 공백·탭 제거.** 줄 끝 공백, 탭, 전각 공백(`　`)은 발음 타이밍을 흔들 수 있다.

## 2. 구두점·호흡 만들기 (한국어 TTS 핵심)

ElevenLabs는 SSML보다 **구두점에 더 민감**하다. 구두점을 호흡 도구로 쓴다.

| 기호 | 효과 | 사용처 |
| --- | --- | --- |
| `,` (쉼표) | 짧은 호흡 (~0.2s) | 절 나누기 |
| `.` (마침표) | 보통 호흡 (~0.4s) | 문장 끝 |
| `…` 또는 `...` | 긴 호흡·여운 (~0.7s) | 극적 정지, 회상 |
| `—` (em-dash) | 중간 호흡 + 약한 강조 | 부연·전환 |
| 빈 줄 한 칸 | 문단 호흡 (~0.8s) | 단락 사이 |
| 빈 줄 두 칸 | 챕터 호흡 (~1.2s) | 섹션 사이 |
| `?` `!` | 억양 변화 | 감정 살리고 싶을 때만 |

### 2.1 권장 패턴
- 긴 문장은 **쉼표로 절을 끊어** 호흡 지점을 만든다.
  - 안 좋음: `프랑크푸르트에서 시작한 사업이 런던과 파리와 빈과 나폴리로 뻗어 나간 것이 가문의 결정적 도약이었다고 정리합니다.`
  - 좋음: `프랑크푸르트에서 시작한 사업이 런던, 파리, 빈, 나폴리로 뻗어 나간 것이 가문의 결정적 도약이었다고 정리합니다.`
- **인용**은 큰따옴표(`"…"`)로 묶고 마침표·쉼표는 따옴표 **앞**에 둔다 (한국어 관행).
- **감정 강조**는 마침표를 줄여 끊는다.
  - 예: `다섯 도시, 다섯 개의 책상.` (긴장감)
- **중성 정보**는 길게 이어도 된다. 다큐 톤은 호흡이 길어야 안정적이다.

### 2.2 SSML / 오디오 태그
- **Multilingual v2**: SSML 태그 **거의 무시**. 구두점만 신뢰.
- **Eleven v3 (alpha)**: 대괄호 오디오 태그 일부 지원. 예: `[whispers]`, `[sighs]`, `[curious]`, `[laughs]`. 한국어 본문 사이에 영어 태그로 끼워 넣는다.
  - 예: `[whispers] 흩어져라. 그러나 떨어지지 마라.`
- `<break time="1.5s" />` 같은 SSML break는 **모델에 따라 무시될 수 있어** 의존하지 않는다. 빈 줄 + 마침표가 더 확실하다.

## 3. 길이·분할 규칙

| 환경 | 권장 1회 길이 | 비고 |
| --- | --- | --- |
| Speech Synthesis (단발 변환) | **약 5,000자 이하** | 무료/유료 공통 안전선 |
| **Studio (장문 프로젝트)** | 챕터당 5,000~10,000자 | 챕터로 잘라 놓고 한 번에 합성 |
| Voice Changer | 문장 단위 권장 | 본 가이드 범위 밖 |

### 3.1 분할 방법
- `로스차일드tts.txt`처럼 **약 1만 자 이상**이면 Studio에 통째로 올리거나, **챕터(빈 줄 두 칸 기준)로 잘라** 여러 파일로 나눈다.
- 분할 시 **문장 중간을 자르지 말 것**. 항상 빈 줄 경계에서만 자른다.
- 챕터별 파일명 예: `로스차일드tts_01.txt`, `로스차일드tts_02.txt` …

### 3.2 토큰·문자 비용
- 한국어 1자 ≒ 1~2 토큰. 1만 자 변환은 대략 1만 5천~2만 캐릭터 크레딧이 소비된다고 보면 안전하다.
- **테스트는 첫 1~2 문단만** 짧게 합성해 톤을 확인한 뒤 전체를 돌린다.

## 4. 보이스 설정 (Voice Settings) 권장값

다큐 내레이션 기준 출발점. 합성 후 들어 보고 ±10~15에서 미세 조정한다.

| 파라미터 | 권장값 | 설명 |
| --- | --- | --- |
| **Stability** | **45~55** | 너무 낮으면 톤 흔들림, 너무 높으면 단조롭다. 다큐는 중간. |
| **Similarity Boost** | **70~80** | 원 보이스의 특성을 강하게 유지. 너무 높이면 잡음 증가. |
| **Style Exaggeration** | **0~15** | 다큐는 거의 0. 드라마틱 회차만 살짝 올림. |
| Speaker Boost | 켬 | 명료도 향상 (Multilingual v2 한정 옵션) |
| Speed (v3) | 1.0 | 한국어 다큐는 0.95~1.0이 자연스럽다 |

### 4.1 보이스 선택 팁 (한국어)
- **한국어 네이티브 보이스**(Voice Library에서 `Korean` 필터)를 1순위로 고른다.
- 다국어 보이스도 한국어 가능하지만 발음 어색한 경우가 있다 — 반드시 **샘플 1문단** 테스트.
- 다큐 내레이션은 **중저음·천천히 말하는 보이스**가 잘 어울린다 (남: 차분한 중년톤 / 여: 안정적인 낭독톤).

## 5. 일레븐랩스 작업 절차 (Studio 기준)

1. **로그인** → ElevenLabs 대시보드 → **Studio** 진입.
2. **New Project** → 제목 입력 (예: `로스차일드 200년`).
3. **Default voice** 선택 → 한국어 보이스 지정 → **Voice Settings** 위 값 적용.
4. **Add chapter** → 챕터 이름(예: `01 도입`) 입력.
5. 본문 영역에 `로스차일드tts.txt` 내용 **복사 붙여넣기** 또는 `.txt` **드래그 업로드**.
6. **Generate** (또는 챕터별 변환) → 미리듣기.
7. 어색한 문장 발견 시 **본문에서 직접 수정**(쉼표 추가, 표기 바꾸기) 후 해당 블록만 재합성.
8. 전체가 OK면 **Export** → MP3/WAV 다운로드.

> 단발 변환이 필요하면 Studio 대신 **Speech Synthesis** 페이지에서 5,000자 단위로 잘라 붙여 넣고 다운로드한다.

## 6. 발음·표기 팁 (한국어 특수 케이스)

ElevenLabs는 한국어 발음이 대체로 자연스럽지만, 아래 경우는 **표기를 살짝 바꿔** 정확도를 올린다.

| 원문 | 추천 표기 | 이유 |
| --- | --- | --- |
| `1812년` | 그대로 OK | 연도 자동 인식 |
| `19세기` | 그대로 OK | |
| `2,000명` | `이천 명` 또는 `2천 명` | 콤마는 무시되거나 어색하게 읽힘 |
| `5%` | `5 퍼센트` | 기호 안전하게 |
| `$100` | `100 달러` | 통화 기호 풀어 쓰기 |
| `WWII` / `2차 대전` | `제2차 세계대전` | 약어보다 풀어 쓰기 |
| `유덴가세` (외국 고유명사) | 그대로, 첫 등장 시 한 번 더 또렷이 발음하고 싶다면 띄어 `유덴 가세`도 가능 | |
| 따옴표 `""` `''` | 한국어 본문에서는 `"…"` 직선 따옴표로 통일하면 안전 | 곡선 따옴표(`""`)는 일부 환경에서 글리치 가능 — `로스차일드tts.txt`는 직선 따옴표로 정리됨 |
| 줄임표 | `...` (점 3개) 또는 `…`(전각) 둘 다 동작. 한 파일 내에서 통일 | |

### 6.1 발음이 자꾸 틀리는 단어가 있을 때
1. **소리 나는 대로** 적는다. 예: `로스차일드` → 정상이지만 다른 외국 이름이 어색하면 `로스차ㅣㄹㄷ` 같은 변형은 **사용하지 말고**, `로스 차일드`처럼 **공백 한 칸** 끼우는 정도가 안전.
2. Studio의 **Pronunciation Dictionary** 기능으로 단어별 IPA/대체 발음을 등록할 수 있다 (Pro 플랜 이상).

## 7. 자주 만나는 오류 / 해결

| 증상 | 원인 | 해결 |
| --- | --- | --- |
| 문장이 이상하게 빠르게 붙어 읽힘 | 빈 줄 부족, 마침표 없음 | 문단 사이 빈 줄, 섹션 제목 끝에 `.` 추가 |
| 중간에 갑자기 톤이 바뀜 | Stability 너무 낮음 | 50 이상으로 올림 |
| 단조롭고 기계적 | Stability 너무 높음 / Style 0 | Stability 40~45로 낮추고 Style 5~10 |
| 외국 이름 발음 어긋남 | 음역 표기 모호 | 공백 한 칸 또는 Pronunciation Dictionary 사용 |
| 따옴표가 "쿼트"로 발음됨 | 직선/곡선 혼용 | 한 종류로 통일 (이 프로젝트는 직선 `"`) |
| 변환이 중간에 끊김 | 1회 한도(약 5,000자) 초과 | Studio 사용하거나 챕터 분할 |
| 한국어 인식이 영어로 잘못 잡힘 | 모델이 Multilingual이 아님 | **Eleven Multilingual v2** 또는 **v3** 선택 확인 |

## 8. 운영 워크플로우 (이 프로젝트 기준)

```
[대본 작성]            [자막 분할]                  [TTS 변환]
대본작성법.md  ──▶  로스차일드.txt  ──복사/병합──▶  로스차일드tts.txt  ──ElevenLabs──▶  MP3
   (서사 원칙)        (큐당 1~2줄)                (자연 문장 + 호흡)
```

1. `대본작성법.md`의 **서사 원칙(0절)**으로 본문을 쓴다.
2. 자막 편집용 **`로스차일드.txt`**를 큐 단위(빈 줄=큐)로 정리한다.
3. **`로스차일드tts.txt`**에는 **자막 줄바꿈을 풀고**, 문장 부호로 호흡을 만든 형태로 옮긴다 (현재 적용 완료).
4. ElevenLabs Studio에 올려 위 4·5절 절차로 합성한다.
5. 다음 회차도 같은 명명 규칙: `<주제>.txt` (자막) / `<주제>tts.txt` (TTS) / 본 가이드를 그대로 적용.

## 9. 업로드 직전 체크리스트

- [ ] UTF-8(BOM 없음) `.txt`로 저장됐는가.
- [ ] Markdown 기호(`#`, `**`, `>`, 백틱)가 본문에 남아 있지 않은가.
- [ ] 자막용 2줄 분할이 모두 풀려 **자연스러운 문장**으로 합쳐졌는가.
- [ ] 섹션 제목 끝에 마침표가 있는가.
- [ ] 따옴표가 한 종류로 통일됐는가 (`"…"` 권장).
- [ ] 1회 합성 한도(약 5,000자) 또는 Studio 챕터 단위에 맞게 **분할 계획**이 있는가.
- [ ] 모델: **Multilingual v2**(또는 v3) 선택했는가.
- [ ] Stability 45~55 / Similarity 70~80 등 **권장값**으로 시작하는가.
- [ ] 첫 1~2 문단으로 **샘플 테스트**를 했는가.

## 10. 프로젝트 파일 정리 (TTS 관련)

| 파일 | 용도 |
| --- | --- |
| `로스차일드.txt` | 자막용 원본. 큐당 1~2줄, 빈 줄=큐 |
| `로스차일드tts.txt` | **ElevenLabs 업로드용**. 자연스러운 문장·문단 호흡 |
| `대본작성법.md` | 서사 원칙·문체·자막 규칙 (작성 단계) |
| `tts.md` | 본 문서 — **TTS 변환 단계** 가이드 |

새 주제는 동일 구조로 만든다: `<주제>.txt`(자막) + `<주제>tts.txt`(TTS).
